**Хаб (hub) (сетевой концентратор) работает на физическом уровне модели OSI (уровень 1)**.

ДОСТОИНСТВА:           
- Дешёвый              
- Простой              

НЕДОСТАТКИ:
- Коллизии
- Низкая безопасность
- Медленный
- Устарел

Пример пакета в хабе:
┌─────────────────────────────────┐
│ FROM: ПК1 (00:11:22:33:44:55)          │
│ TO:   ПК3 (00:11:22:33:44:77)             │
│ DATA: "Привет ПК3!"                         │
└─────────────────────────────────┘

Хаб: "Эй, все! Пакет от ПК1 для ПК3!"
Получают: ПК1, ПК2, ПК3... (все устройства)

**Коммутатор (свитч) работает на канальном уровне модели OSI** (Open Systems Interconnection). 

ДОСТОИНСТВА:                 
- Высокая скорость           
- Безопасность               
- Нет коллизий
- Умная доставка

НЕДОСТАТКИ:
- Дороже хаба
- Сложнее в настройке

Пример в коммутаторе:
MAC-таблица:
┌─────────────────────────────┐
│ MAC Address      │ Port               │
├─────────────────────────────┤
│ 00:11:22:33:44:55│    1                 │ ← ПК1
│ 00:11:22:33:44:77│    3                 │ ← ПК3
│ 00:11:22:33:44:88│    2                 │ ← ПК2
└─────────────────────────────┘

Коммутатор: "Пакет для 00:11:22:33:44:77 → отправляю на порт 3"
Получает: ТОЛЬКО ПК3

**Маршрутизатор (роутер) работает на сетевом уровне модели OSI** (Open Systems Interconnection). **Маршрутизатор (роутер) работает как «навигационная система» сети**. 

ДОСТОИНСТВА:                 
- Соединение сетей           
- NAT, Firewall              
- DHCP, DNS                  
- Безопасность

 НЕДОСТАТКИ:
  - Самый дорогой
  - Сложная настройка
  - Требует знаний

Пример в маршрутизаторе:
Таблица маршрутизации:
┌─────────────────────────────────────┐
│ Network                │ Interface      │ Metric │
├─────────────────────────────────────┤
│ 192.168.1.0/24      │   LAN            │    0      │
│ 10.0.0.0/8              │   WAN          │    1      │
│ 0.0.0.0/0                │   WAN          │    10    │
└─────────────────────────────────────┘

Маршрутизатор:  "Пакет для 8.8.8.8 → не моя сеть → отправляю на WAN"
