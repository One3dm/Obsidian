**Хаб (hub) (сетевой концентратор) работает на физическом уровне модели OSI (уровень 1)**.

ДОСТОИНСТВА:           
- Дешёвый              
- Простой              

НЕДОСТАТКИ:
- Коллизии
- Низкая безопасность
- Медленный
- Устарел

Пример пакета в хабе:
┌─────────────────────────────────┐
│ FROM: ПК1 (00:11:22:33:44:55)          │
│ TO:   ПК3 (00:11:22:33:44:77)             │
│ DATA: "Привет ПК3!"                         │
└─────────────────────────────────┘

Хаб: 📢 "Эй, все! Пакет от ПК1 для ПК3!"
Получают: ПК1, ПК2, ПК3... (все устройства)

**Коммутатор (свитч) работает на канальном уровне модели OSI** (Open Systems Interconnection). 

ДОСТОИНСТВА:                 
- Высокая скорость           
- Безопасность               
- Нет коллизий
- Умная доставка

НЕДОСТАТКИ:
- Дороже хаба
- Сложнее в настройке

Пример в коммутаторе:
MAC-таблица:
┌─────────────────────────────┐
│ MAC Address      │ Port               │
├─────────────────────────────┤
│ 00:11:22:33:44:55│    1                 │ ← ПК1
│ 00:11:22:33:44:77│    3                 │ ← ПК3
│ 00:11:22:33:44:88│    2                 │ ← ПК2
└─────────────────────────────┘

Коммутатор:"Пакет для 00:11:22:33:44:77 → отправляю на порт 3"
Получает: ТОЛЬКО ПК3

**Маршрутизатор (роутер) работает на сетевом уровне модели OSI** (Open Systems Interconnection). **Маршрутизатор (роутер) работает как «навигационная система» сети**. Он получает, обрабатывает и перенаправляет пакеты данных, определяя, куда именно они должны попасть. Это позволяет:
- **Соединять разные сети** — например, локальную сеть с интернетом.
- **Создавать локальную сеть**, объединяя устройства между собой. Это позволяет, например, передавать файлы с ноутбука на принтер или смотреть видео с домашнего медиа сервера.
